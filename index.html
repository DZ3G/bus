Bienvenue. Aujourd'hui on parle du bus CAN — Controller Area Network — inventé par Bosch en 1986 pour l'automobile. Depuis, il est devenu un standard industriel mondial.

L'animation montre les deux fils différentiels CAN-H et CAN-L. Quand un bit est dominant (logique 0), CAN-H monte à ~3,5V et CAN-L descend à ~1,5V. Cette logique différentielle rend le bus très robuste au bruit électromagnétique.

2 fils différentiels : CAN-H et CAN-L. La communication différentielle annule les interférences — parfait dans un environnement bruité comme une voiture ou une usine.

8 Mbps : en mode CAN FD (2012). Le CAN classique est limité à 1 Mbps. La vitesse diminue avec la longueur du câble : 1 Mbps = 40m max, 125 kbps = 500m.

128 nœuds : tous reçoivent tous les messages, chacun filtre selon l'identifiant.

1986 : Bosch développe CAN pour remplacer les harnais électriques dans les Mercedes. Aujourd'hui, chaque voiture embarque en moyenne 70 nœuds CAN.

01 — Multi-maître : Tous les nœuds peuvent initier une transmission à tout moment. Il n'y a pas de maître désigné. N'importe quel nœud peut envoyer un message quand le bus est libre.

Rouge — SOF / EOF : Start Of Frame (1 bit dominant) annonce la trame. End Of Frame (7 bits recessifs) la termine.

Bleu — ID (11 bits) : identifiant du message et sa priorité. Plus l'ID est petit, plus la priorité est haute. CAN étendu (2.0B) utilise 29 bits.

Violet — RTR / DLC : RTR distingue trame de données (0) vs requête (1). DLC indique le nombre d'octets (0–8 classique, 0–64 FD).

Jaune — CRC : 15 bits de contrôle d'erreur recalculés par chaque récepteur.

Vert — ACK : tous les récepteurs corrects tirent ACK à dominant pour confirmer la réception.

Versions & Vitesses

CAN classique (2.0A/B) : limité à 1 Mbps et 8 octets par trame. Suffisant pour des capteurs lents. Standard dans les voitures depuis les années 90.

CAN FD (2012) : même arbitrage à 1 Mbps, mais la phase données monte à 8 Mbps. 64 octets par trame. Obligatoire dans les nouvelles voitures.

CAN XL (2023) : 2048 octets par trame, 10 Mbps. Conçu pour les ECU haute performance. ISO 11898-1:2024.

Règle : 1 Mbps = max 40m, 500 kbps = 100m, 125 kbps = 500m.

Implémentation

Exemple Arduino

C++ · Arduino + MCP2515 (SPI→CAN) 
// Envoi d'une trame CAN — MCP2515 via SPI
#include <SPI.h>
#include <mcp2515.h>   // Library autowp/arduino-mcp2515

MCP2515 mcp2515(10);  // CS pin = 10

void setup() {
  Serial.begin(115200);
  SPI.begin();
  mcp2515.reset();
  mcp2515.setBitrate(CAN_500KBPS, MCP_8MHZ);
  mcp2515.setNormalMode();
  Serial.println("CAN Bus prêt");
}

void loop() {
  // ── ENVOI ──────────────────────────────
  struct can_frame txFrame;
  txFrame.can_id  = 0x123;   // ID 11 bits = priorité
  txFrame.can_dlc = 4;       // 4 octets de données
  txFrame.data[0] = 0xDE;  txFrame.data[1] = 0xAD;
  txFrame.data[2] = 0xBE;  txFrame.data[3] = 0xEF;
  mcp2515.sendMessage(&txFrame);

  // ── RÉCEPTION ──────────────────────────
  struct can_frame rxFrame;
  if(mcp2515.readMessage(&rxFrame) == MCP2515::ERROR_OK){
    Serial.print("ID: 0x"); Serial.println(rxFrame.can_id, HEX);
    for(int i=0; i<rxFrame.can_dlc; i++)
      Serial.print(rxFrame.data[i], HEX);
    Serial.println();
  }
  delay(100);
}

L'Arduino n'a pas de contrôleur CAN intégré. On utilise le MCP2515 (Microchip) connecté en SPI, couplé au transceiver TJA1050 pour le niveau physique CAN-H/CAN-L.

can_id = 0x123 : identifiant 11 bits. Tous les nœuds voient la trame et filtrent selon l'ID.

can_dlc = 4 : 4 octets de données. CRC et ACK sont gérés automatiquement en hardware par le MCP2515.

Pour ESP32 ou STM32 : contrôleur CAN souvent intégré, seul le transceiver externe est nécessaire.
